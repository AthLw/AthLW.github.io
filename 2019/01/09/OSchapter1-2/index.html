操作系统

chapter1 操作系统概论

操作系统定义：一组用于控制和管理计算机资源的程序集合

计算机资源分为两类：

  1. 硬件资源：处理器，设备，存储器等
  2. 信息资源：程序和数据

1.1 操作系统功能

  1. 处理器管理
  2. 存储管理
  3. 设备管理
  4. 文件管理
  5. 网络管理

1.2 操作系统的特性

  1. 并发性
    指多个事件或活动在同一时间间隔内发生

    并行：指多个事件或活动在同一时刻发生

  2. 共享性
    指资源可以被多个并发程序共同使用

    并发和共享是操作系统两个最基本的特性，它们相互依存

  3. 不确定性（异步性）
    不确定性是并发与共享的必然结果

  4. 虚拟性

1.3 操作系统的发展

  1. 手工阶段
  2. 批处理阶段

  3. 执行处理阶段
    通道技术和中断技术出现

  4. 多道处理阶段
    多个程序同时装入内存并启动交替执行
    多道运行的特征： 

     1. 多道，多道程序同时在内存中
 2. 宏观上并行
 3. 微观上串行

1.4 操作系统分类

  1. 批处理操作系统
  2. 分时操作系统
  3. 实时操作系统
  4. 网络操作系统
  5. 分布式操作系统
  6. 嵌入式操作系统

chapter2 用户界面

操作系统通过 程序接口和操作接口 两种方式提供服务和功能

2.1 操作接口

又称 命令接口

提供的命令有：

  1. 系统访问类
  2. 文件操作
  3. 磁盘操作命令
  4. 目录操作命令
  5. 其他命令

2.2 系统调用

目的：扩充机器能力，增强系统能力，方便用户使用

系统调用本质上也是一种特殊的过程调用，但也有差别：

  1. 调用形式不同：过程使用一般调用指令，而系统调用按功能号调用
  2. 被调用代码位置不同：过程函数是静态调用，经过编译连接后不可改变。而系统调用是动态调用，代码在程序之外。
  3. 提供方式不同：过程函数有编译系统提供，而系统调用有操作系统提供。
  4. 调用实现方式不同：过程通过跳转指令，而系统调用通过中断机构实现。

系统调用分类（了解）：

  1. 进程控制类
  2. 进程通信类
  3. 文件操作类
  4. 设备管理类
  5. 内存管理类
  6. 信息维护类
