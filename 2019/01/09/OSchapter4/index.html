操作系统复习

操作系统第四章

进程关系

同步和互斥，互斥是同步的特例

互斥是相同进程之间，又称间接制约
同步是不同进程之间，又称直接制约

临界区

使用临界区原则：

  1. 有空让进
  2. 无空等待
  3. 多中择一
  4. 有限等待
  5. 让权等待

信号量与PV操作

  1. 生产者消费者问题

  2. 苹果橘子问题

  3. 读者写者问题

进程间通信

进程交互基本要求：同步和通信

通信方式：

  1. 主从式
  2. 会话式
  3. 消息队列和信箱式
  4. 共享存储区式

管道本质是一个共享文件

死锁

  1. 死锁原因

    • 资源不足
    • 分配不当

    • 进程推进顺序不合适

      死锁的4个必要条件：

      1. 互斥使用
      2. 不可剥夺
      3. 请求和保持
      4. 循环等待

  2. 死锁预防

    静态方法，破坏四个必要条件中其中一个

    • 破坏不可剥夺
    • 破坏请求和保持
    • 破坏循环等待

  3. 死锁避免

    动态方法
    银行家算法

  4. 死锁检测

    资源分配图

    资源分配图有环路是死锁的 必要条件
    死锁的充分条件是：资源分配图不可完全简化

  5. 死锁的解除

    1. 资源剥夺法
    2. 撤销进程法

管程

一个数据结构和能为并发进程所执行的一组操作。
管程是集中式同步进程，用共享数据结构抽象表示系统中的共享资源
